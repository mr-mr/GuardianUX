Statistics module

Live scores, fixtures and results

  • All fixtures - shows all fixtures for the coming week ahead. The user has the ability to filter this list by nation/competition (the filter doesn't work, but it should be obvious what it does), and also to load in another weeks worth of fixtures when they reach the bottom of the page.

Football section front

These are using the scenario of a Saturday during the premiership season. They should cover enough bases for application to other days of the week/competitions.

There are some very basic rules on what is shown where within the list of games. The priority order should be Live, fixtures then results. This allows live games to always be at the top of the list. I've explained this in more detail below.

  • Football section front on the morning of matchday - all games are upcoming fixtures so they appear in the list as normal.
  • Football section front during lunchtime kick off - the first fixture in the list becomes live, with the score linking to the live blog for this game. The kick off time changes to indicate which stage the game is at (1st, HT, 2nd, ET, PEN, FT).
  • Football section front during 3pm kick offs - the Chelsea game is now a result, and so drops to the end of the list, allowing the 3pm kick offs, now live, to be shown first. The scores for the 3pm kick offs link to the multi game live blog, and kick off times change as above.
  • Football section front at 5pm - The 3pm kick offs have finished, but the 5.30 game has yet to start. As the 3pm games are now results, they drop to the bottom of the list, and the 5pm fixture is shown top as there are no other live games.
  • Football section front during 5.30pm kick off - The final fixture of the day now becomes live, with the score linking to the live blog for this game, and the kick off time changing as above.
  • Football section front evening/night - All games have been played and are now results. This should shown at the top for a period of time, before dropping down the section front.
  • Football section front the morning of FA Cup third round - This is to show what it would look like if we included all the major fixtures for the day. FA Cup third round is the most fixtures we'll have, with 32 (and that's assuming all play on the Saturday which wont happen as they're usually spread over the weekend).
  • Football section front showing multiple competition fixtures - later on in the season when the FA Cup and Champions League reach their latter stages, league games or Cup replays can be played alongside other these competitions. The acronym to the left indicates competition.

    • The live/fixtures/results module is intended to be flexibile in it's positioning, i.e. it shouldn't always be top. My initial thoughts are that it should be top of the page from the get go on Saturday's, somewhere further down initially on Sundays - Thursdays. On Sunday it should go top at around 11am and Mon-Thurs it should move to the top around 2pm/3pm. Fridays I don't think there are any significant fixtures (i.e. Prem, FA Cup, UEFA), though there might be the odd FA Cup game.

    It would be good if the positioning were automated, with an overide (i.e. if something significant happens, player dies etc, we can lead with that, rather than the fixtures module).

Live blogs/pages

Slight update here as I've removed the pull to refresh version, which will be replaced presently. Stats have been massaged to show 50/50 styling and any tables and galleries are placeholders whilst we find out if we can do them (more so the gallery - this is intended to be automated from whatever picture provider Guardian uses.).

  • Version one - Auto updating, with switch to turn off. If a user turns it off a notification appears showing the amount of updates that have been missed. The last 20 minutes (arbitary, we could make it ten or fifteen) are shown by default, with a 'load more' option which loads in the previous 20 minutes and so on. Finally, the score will appear if you scroll down, fixed to the top. The idea here is that this live updates as well, so you can see when a goal has gone in if you are halfway down the page.
  • Version two - as above but for multiple games. The scores are shown when you scroll down the page, but cylce between all games – this aspect is a bit shakey, but you get the idea (it's supposed to reset to the first fixture if you scroll right to the top and scroll down again, but it doesn't quite work). You can also see more detail (i.e. scorers and cards) on the scores tab.
  • Version three - single game, with pull to refresh. No sticky scores when you scroll on this version as it interferes with the pull to refresh from a UX perspective when you scroll up and down. Statistics and the table are off canvas, revealed with either taps/clicks or swipes left/right. This obviously only works if we don't use this method for the navigation.
  • Version four - as above but for multiple games. Statistics is replaced by scores

Navigation

Three variants on navigation

  • Navigation test one - pretty much as now. I've kept similar positioning, but the two triggers to reveal the navigation could equally be icons positioned on the right, level with the logo.
  • Navigation test two - all navigation positioned after the content (probably using JS for this as it will more than likely need to be at the top for larger screens). The good thing about this is that placing it here gives a user options at the end of every page. As pervious, the two anchor links at the top could be icons or text in the final design.
  • Navigation test three - riffing off the Financial Times app and what people like the Boston Globe and The Times are doing. The sections/search areas still open up, but the sections navigation also surfaces content, whilst the search navigation shows recent search terms. This would probably have to be contextual, so as you navigated deeper into the site, the sections would change to, for example, the sport sections (with the link to show/hide the sections changing to 'sport sections' to reflect this).
  • Navigation test four - off canvas, Facebook/Twitter app style navigation. This should also be swipeable (left/right), might be a bit buggy. The fallback for no JS support/less capable browsers would be to have the navigation at the bottom (similar to version two basically). This demo is really quick and dirty so there are a few things wrong with it, such as content and nav butting up together, the trigger button being on the right rather than the left etc.

Caveats

As ever, there's a few things to bear in mind:

  • There are no, repeat no, adverts on the experiment (or any other similar content). Clearly this isn't going to be the case with the finished result – it was merely a time saving measure.
  • Adverts on the current versions might either not appear or appear a bit odd.
  • It'll only work in Chrome and Safari, probably Firefox/Fennec and possibly Opera/Opera mini.
  • This isn't, and I can't stress this enough, a design. The styling and 'design' of it is rudimentary and it's intended as an experimental wireframe that we can tear apart and nothing more.
